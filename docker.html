الأساسيات

المفاهيم الأساسية

قبل كتابة أول أمر، تعرّف على اللبنات الأساسية لـ Docker

📦
Container
الحاوية هي بيئة تشغيل معزولة تحتوي على التطبيق وكل ما يحتاجه. خفيفة، سريعة، وتعمل بنفس الطريقة في كل مكان — سواء على جهازك أو السيرفر.
🖼️
Image
الصورة هي قالب للقراءة فقط يُستخدم لإنشاء الحاويات. تحتوي على نظام التشغيل، المكتبات، والتطبيق. مثل ISO القرص القابل للتثبيت.
📄
Dockerfile
ملف نصي يحتوي على تعليمات لبناء Image مخصصة. يحدد الصورة الأساسية، الملفات، والأوامر — مثل وصفة لإنشاء صورتك.
🗄️
Volume
آلية لحفظ البيانات بشكل دائم خارج الحاوية. بدونه، تُفقد البيانات عند حذف الحاوية. ضروري لقواعد البيانات.
🌐
Network
الشبكة تربط الحاويات ببعضها أو بالعالم الخارجي. الحاويات على نفس الشبكة تتواصل باسمها مباشرة.
📋
Docker Compose
أداة لتعريف وتشغيل تطبيقات متعددة الحاويات. ملف YAML واحد يصف جميع الخدمات — شغّل كل شيء بأمر واحد.
🏪
Registry
مستودع لتخزين ومشاركة الصور. Docker Hub هو المستودع الافتراضي. يمكنك رفع صورك وتنزيل الصور الجاهزة (Nginx, MySQL, Redis...).
🏗️
Layer
كل تعليمة في Dockerfile تُنشئ طبقة جديدة. Docker يخزن الطبقات ويُعيد استخدامها (cache) لتسريع عمليات البناء وتوفير المساحة.
